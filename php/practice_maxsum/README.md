# 部分配列の最大和問題

## 問題

```php
function solution($nums) {
}
```

`solution` 関数には int 型の配列 `$nums` が引数として与えられます。

```
Array
(
    [0] => -2
    [1] => 1
    [2] => -3
    [3] => 4
    [4] => -1
    [5] => 2
    [6] => 1
    [7] => -5
    [8] => 4
)
```

配列 `$nums` の中から、**連続する部分配列**を選び、その要素の合計が最大となる値を求めてください。

部分配列は少なくとも1つの要素を含む必要があります。

## 例

### 例1
```
入力 : [-2, 1, -3, 4, -1, 2, 1, -5, 4]
出力 : 6
```

**解説**: 部分配列 `[4, -1, 2, 1]` の合計が `6` で最大です。

### 例2
```
入力 : [1]
出力 : 1
```

**解説**: 要素が1つだけなので、その値が答えです。

### 例3
```
入力 : [5, 4, -1, 7, 8]
出力 : 23
```

**解説**: 配列全体 `[5, 4, -1, 7, 8]` の合計が `23` で最大です。

### 例4
```
入力 : [-1, -2, -3, -4]
出力 : -1
```

**解説**: すべて負の数の場合、最も大きい `-1` が答えです。

### 例5
```
入力 : [1, 2, 3, -6, 1, 2, 3]
出力 : 6
```

**解説**: `[1, 2, 3]`（前半）または `[1, 2, 3]`（後半）の合計が `6` で最大です。

## 制約

```
1 <= len(nums) <= 10000
-10000 <= nums[i] <= 10000
```

## ヒント

この問題は「最大部分配列問題」と呼ばれる典型問題です。

各位置で「ここまでの部分配列の最大和」を考えると、効率的に解くことができます。

現在の要素を新しい部分配列の開始点にするか、既存の部分配列に追加するか、どちらが得かを考えてみましょう。
