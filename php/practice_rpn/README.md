# 逆ポーランド記法の計算問題

## 問題

```php
function solution($tokens) {
}
```

`solution` 関数には、**逆ポーランド記法（RPN: Reverse Polish Notation）** で表された数式が配列 `$tokens` として与えられます。

逆ポーランド記法とは、演算子を被演算子の**後ろ**に置く記法です。

```
通常の記法: 3 + 4
逆ポーランド記法: 3 4 +
```

配列 `$tokens` には、整数または演算子（`+`, `-`, `*`, `/`）が含まれます。

この数式を計算し、結果を整数で返してください。

**注意:**
- 除算は **0に向かって切り捨て** とします（例: `-7 / 2 = -3`）
- 演算子は必ず2つの被演算子を取ります
- 入力は常に有効な逆ポーランド記法であることが保証されています

## 例

### 例1
```
入力 : ["2", "1", "+", "3", "*"]
出力 : 9
```

**解説**: `((2 + 1) * 3) = 9`

### 例2
```
入力 : ["4", "13", "5", "/", "+"]
出力 : 6
```

**解説**: `(4 + (13 / 5)) = (4 + 2) = 6`

### 例3
```
入力 : ["10", "6", "9", "3", "+", "-11", "*", "/", "*", "17", "+", "5", "+"]
出力 : 22
```

**解説**:
```
  ((10 * (6 / ((9 + 3) * -11))) + 17) + 5
= ((10 * (6 / (12 * -11))) + 17) + 5
= ((10 * (6 / -132)) + 17) + 5
= ((10 * 0) + 17) + 5
= 22
```

### 例4
```
入力 : ["3", "4", "+"]
出力 : 7
```

**解説**: `3 + 4 = 7`

### 例5
```
入力 : ["5"]
出力 : 5
```

**解説**: 数字のみの場合、その値をそのまま返します。

## 制約

```
1 <= len(tokens) <= 10000
tokens[i] は整数を表す文字列、または "+", "-", "*", "/" のいずれか
-1000 <= 整数値 <= 1000
ゼロ除算は発生しない
```

## ヒント

スタックを使って、以下のように処理します：

1. トークンを先頭から順に処理する
2. 数字ならスタックに積む（push）
3. 演算子なら、スタックから2つ取り出し（pop）、計算結果をスタックに積む
4. 最後にスタックに残った値が答え

**ポイント**: 演算子の場合、先に pop した値が「右側」の被演算子になります。
