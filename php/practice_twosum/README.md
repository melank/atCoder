# 2つの数の和問題

## 問題

```php
function solution($nums, $target) {
}
```

`solution` 関数には、**昇順にソートされた** 整数配列 `$nums` と、目標値 `$target` が引数として与えられます。

配列の中から **2つの異なる要素** を選び、その和が `$target` と等しくなるペアを見つけてください。

ペアが存在する場合は、その **2つのインデックス** を配列 `[index1, index2]` として返してください（`index1 < index2`）。

ペアが存在しない場合は `[-1, -1]` を返してください。

**注意:**
- 同じ要素を2回使うことはできません
- 答えが複数ある場合は、どれか1つを返せばOKです

## 例

### 例1
```
入力 : nums = [1, 2, 3, 4, 5], target = 9
出力 : [3, 4]
```

**解説**: `nums[3] + nums[4] = 4 + 5 = 9`

### 例2
```
入力 : nums = [2, 7, 11, 15], target = 9
出力 : [0, 1]
```

**解説**: `nums[0] + nums[1] = 2 + 7 = 9`

### 例3
```
入力 : nums = [1, 2, 3, 4, 5], target = 100
出力 : [-1, -1]
```

**解説**: 和が100になるペアは存在しません。

### 例4
```
入力 : nums = [-3, -1, 0, 2, 4, 6], target = 3
出力 : [1, 4]
```

**解説**: `nums[1] + nums[4] = -1 + 4 = 3`

### 例5
```
入力 : nums = [1, 1, 2, 3], target = 2
出力 : [0, 1]
```

**解説**: `nums[0] + nums[1] = 1 + 1 = 2`

## 制約

```
2 <= len(nums) <= 10000
-10000 <= nums[i] <= 10000
-20000 <= target <= 20000
nums は昇順にソートされている
```

## ヒント

この問題は「2ポインタ法（Two Pointers）」で効率的に解けます。

1. 配列の **先頭** と **末尾** にポインタを置く
2. 2つの要素の和が `target` より小さければ、左のポインタを右に動かす
3. 2つの要素の和が `target` より大きければ、右のポインタを左に動かす
4. 一致すれば答え

配列がソート済みであることを活かしましょう！
