# 会議室スケジューリング問題

## 問題

```php
function solution($meetings) {
}
```

`solution` 関数には int 型の2次元配列である `$meetings` が引数として与えられます。`$meetings` の各要素には、`[start, end]` のように会議の開始時刻と終了時刻を示します。

```
Array
(
    [0] => Array
        (
            [0] => 1
            [1] => 3
        )

    [1] => Array
        (
            [0] => 2
            [1] => 4
        )

    [2] => Array
        (
            [0] => 3
            [1] => 5
        )

)
```

あなたは1人で複数の会議に参加したいと考えています。ただし、**1つの会議に参加中は他の会議に参加できません**。会議の終了時刻と別の会議の開始時刻が同じ場合は、両方の会議に参加可能です。

**参加できる最大の会議数**を返す `solution` 関数を実装してください。

## 例

### 例1
```
入力 : [[1, 3], [2, 4], [3, 5]]
出力 : 2
```

**解説**: [1, 3] と [3, 5] の2つの会議に参加できます。[1, 3] が終了する時刻3に [3, 5] が開始するため、両方に参加可能です。

### 例2
```
入力 : [[1, 2], [2, 3], [3, 4], [1, 4]]
出力 : 3
```

**解説**: [1, 2], [2, 3], [3, 4] の3つの会議に参加できます。[1, 4] を選ぶと1つしか参加できません。

### 例3
```
入力 : [[7, 9], [0, 3], [2, 5], [4, 6]]
出力 : 3
```

**解説**: [0, 3], [4, 6], [7, 9] の3つの会議に参加できます。

### 例4
```
入力 : [[1, 10]]
出力 : 1
```

**解説**: 会議が1つだけなので、1つに参加できます。

## 制約

```
1 <= len(meetings) <= 5000
任意の会議 meetings[i] は常に長さが 2 となる
0 <= start < end <= 10000 が満たされる
```

## ヒント

この問題は「区間スケジューリング問題」と呼ばれる典型問題です。貪欲法（Greedy Algorithm）を使って解くことができます。

どの順番で会議を選んでいくと最適になるか考えてみましょう。
