# 長さKの部分配列の最大平均値

## 問題

```php
function solution($nums, $k) {
}
```

整数配列 `$nums` と整数 `$k` が与えられます。

**連続する長さ `k` の部分配列** の中で、**平均値が最大** となるものを見つけ、その平均値を返してください。

## 例

### 例1
```
入力 : nums = [1, 12, -5, -6, 50, 3], k = 4
出力 : 12.75
```

**解説**:
- `[12, -5, -6, 50]` の平均 = (12 + (-5) + (-6) + 50) / 4 = 51 / 4 = 12.75

### 例2
```
入力 : nums = [5], k = 1
出力 : 5.0
```

**解説**: 配列に1要素しかなく、k=1 なので、その値が答え。

### 例3
```
入力 : nums = [1, 2, 3, 4, 5], k = 3
出力 : 4.0
```

**解説**:
- `[1, 2, 3]` の平均 = 2.0
- `[2, 3, 4]` の平均 = 3.0
- `[3, 4, 5]` の平均 = **4.0** ← 最大

### 例4
```
入力 : nums = [0, 1, 1, 3, 3], k = 4
出力 : 2.0
```

**解説**:
- `[0, 1, 1, 3]` の平均 = 5 / 4 = 1.25
- `[1, 1, 3, 3]` の平均 = 8 / 4 = **2.0** ← 最大

### 例5
```
入力 : nums = [-1, -2, -3, -4, -5], k = 2
出力 : -1.5
```

**解説**: すべて負の数でも、最大の平均を求める。

## 制約

```
1 <= k <= len(nums) <= 10000
-10000 <= nums[i] <= 10000
```

## ヒント

この問題は「スライディングウィンドウ」で効率的に解けます。

### 単純な方法（遅い）
すべての長さkの部分配列について和を計算 → O(N × K)

### スライディングウィンドウ（速い）
1. 最初のk個の和を計算
2. ウィンドウを1つずつスライド: **左端を引いて、右端を足す**
3. 各ステップで最大和を更新

```
配列: [1, 12, -5, -6, 50, 3], k=4

[1, 12, -5, -6] → 和 = 2
   [12, -5, -6, 50] → 和 = 2 - 1 + 50 = 51 ← 最大
       [-5, -6, 50, 3] → 和 = 51 - 12 + 3 = 42
```

O(N) で解ける！
