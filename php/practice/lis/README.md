# 最長増加部分列（LIS: Longest Increasing Subsequence）

## 問題

整数配列が与えられます。**狭義単調増加**する部分列のうち、最も長いものの長さを求めてください。

部分列とは、元の配列から一部の要素を取り出して順序を保ったものです（連続していなくてもOK）。

## 関数シグネチャ

```php
function solution(array $nums): int
```

## 例

### 例1
```
入力: nums = [10, 9, 2, 5, 3, 7, 101, 18]
出力: 4
説明: 最長増加部分列は [2, 3, 7, 18] または [2, 3, 7, 101] で、長さは4
```

### 例2
```
入力: nums = [0, 1, 0, 3, 2, 3]
出力: 4
説明: 最長増加部分列は [0, 1, 2, 3]
```

### 例3
```
入力: nums = [7, 7, 7, 7, 7]
出力: 1
説明: すべて同じ値なので、増加部分列は1要素のみ
```

### 例4
```
入力: nums = [1, 2, 3, 4, 5]
出力: 5
説明: 全体が増加列
```

### 例5
```
入力: nums = [5, 4, 3, 2, 1]
出力: 1
説明: 減少列なので、増加部分列は1要素のみ
```

## 制約

- `1 <= len(nums) <= 2500`
- `-10000 <= nums[i] <= 10000`

## ヒント

### O(N²) 解法
- `dp[i]` = `nums[i]` で終わる最長増加部分列の長さ
- 各 `i` に対して、`j < i` かつ `nums[j] < nums[i]` なら `dp[i] = max(dp[i], dp[j] + 1)`

### O(N log N) 解法（発展）
- 長さ `k` の増加部分列の末尾の最小値を管理
- 二分探索で更新位置を決定
