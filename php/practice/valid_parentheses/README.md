# 有効な括弧（Valid Parentheses）

## 難易度
★★☆☆☆（初級）

## アルゴリズム
スタック

## 問題

`()`、`{}`、`[]` の3種類の括弧のみで構成された文字列 `s` が与えられます。

この文字列が**有効な括弧列**かどうかを判定してください。

有効な括弧列とは：
1. 開き括弧は、対応する同じ種類の閉じ括弧で閉じられている
2. 開き括弧は、正しい順序で閉じられている
3. 各閉じ括弧には、対応する同じ種類の開き括弧がある

## 入出力例

### 例1
```
入力: s = "()"
出力: true
```

### 例2
```
入力: s = "()[]{}"
出力: true
説明: 3組の括弧がそれぞれ正しく閉じている
```

### 例3
```
入力: s = "(]"
出力: false
説明: ( に対して ] で閉じようとしている
```

### 例4
```
入力: s = "([)]"
出力: false
説明: ( を閉じる前に [ を閉じようとしている
```

### 例5
```
入力: s = "{[]}"
出力: true
説明: 内側の [] が先に閉じ、外側の {} が後に閉じる
```

## 制約

- 0 <= len(s) <= 10^4
- s は `()[]{}` のみで構成される

## ヒント

### スタックを使う理由
- 最後に開いた括弧を最初に閉じる必要がある（LIFO）
- これはスタックの特性と一致する

### アルゴリズム
1. 空のスタックを用意
2. 文字列を1文字ずつ走査
3. 開き括弧なら → スタックに push
4. 閉じ括弧なら → スタックから pop して対応を確認
5. 最後にスタックが空なら有効

### 注意点
- 閉じ括弧が来たときスタックが空なら無効
- 走査終了時にスタックに残りがあれば無効
