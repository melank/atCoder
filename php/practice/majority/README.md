# 多数決要素（Majority Element）

## 問題

```php
function solution(array $nums): int {
}
```

整数配列 `$nums` が与えられます。

配列の中で**過半数**（n/2 回より多く）出現する要素を返してください。

過半数要素は必ず存在すると仮定します。

## 例

### 例1
```
入力: [3, 2, 3]
出力: 3
```

**解説**: 3 が 2回出現（過半数）

### 例2
```
入力: [2, 2, 1, 1, 1, 2, 2]
出力: 2
```

**解説**: 2 が 4回出現（7/2 = 3.5 より多い）

### 例3
```
入力: [1]
出力: 1
```

### 例4
```
入力: [1, 1, 1, 1]
出力: 1
```

### 例5
```
入力: [6, 5, 5]
出力: 5
```

## 制約

```
1 <= count($nums) <= 10000
過半数要素は必ず存在する
```

## ヒント

### アプローチ1: ハッシュマップ（O(N) 時間, O(N) 空間）

各要素の出現回数をカウント

### アプローチ2: Boyer-Moore 投票アルゴリズム（O(N) 時間, O(1) 空間）

1. **候補**と**カウント**を管理
2. カウントが 0 なら、現在の要素を候補にする
3. 現在の要素が候補と同じならカウント++、違えばカウント--
4. 最後に残った候補が過半数要素

### Boyer-Moore の直感

過半数要素は半分以上を占めるので、他の全要素と「相殺」しても必ず残る。

```
nums = [2, 2, 1, 1, 1, 2, 2]

i=0: candidate=2, count=1
i=1: candidate=2, count=2  (2==2)
i=2: candidate=2, count=1  (1!=2)
i=3: candidate=2, count=0  (1!=2)
i=4: candidate=1, count=1  (count==0 なので更新)
i=5: candidate=1, count=0  (2!=1)
i=6: candidate=2, count=1  (count==0 なので更新)

結果: 2
```

### 計算量

| アプローチ | 時間 | 空間 |
|-----------|------|------|
| ハッシュマップ | O(N) | O(N) |
| Boyer-Moore | O(N) | O(1) |
