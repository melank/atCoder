# 最近のリクエスト数

**難易度**: ★★☆☆☆（初級）

**カテゴリ**: データ構造（キュー）

## 問題

あなたは Web サーバーのリクエスト監視システムを作成しています。

サーバーには時系列順にリクエストが届きます。各リクエストにはタイムスタンプ（ミリ秒単位の整数）が付与されています。

各リクエストが届いたとき、「過去 3000 ミリ秒以内（そのリクエスト自身を含む）に届いたリクエストの数」を返してください。

つまり、タイムスタンプ `t` のリクエストが届いたとき、`[t - 3000, t]` の範囲にあるリクエストの数を数えます。

リクエストのタイムスタンプは必ず昇順（単調非減少）で与えられます。

## 入力

```
N
t_1
t_2
...
t_N
```

- 1 行目: リクエストの数 `N`
- 2 行目以降: 各リクエストのタイムスタンプ `t_i`（昇順）

## 出力

各リクエストに対して、過去 3000 ミリ秒以内のリクエスト数を空白区切りで 1 行で出力してください。

## 制約

- `1 <= N <= 10^4`
- `1 <= t_i <= 10^9`
- タイムスタンプは昇順（`t_i <= t_{i+1}`）

## 入出力例

### 例 1

**入力**
```
4
1
100
3001
3002
```

**出力**
```
1 2 3 3
```

**説明**
- `t=1`: 範囲 `[-2999, 1]` にあるリクエストは `{1}` の 1 個
- `t=100`: 範囲 `[-2900, 100]` にあるリクエストは `{1, 100}` の 2 個
- `t=3001`: 範囲 `[1, 3001]` にあるリクエストは `{1, 100, 3001}` の 3 個
- `t=3002`: 範囲 `[2, 3002]` にあるリクエストは `{100, 3001, 3002}` の 3 個（`t=1` は範囲外）

### 例 2

**入力**
```
1
1000000000
```

**出力**
```
1
```

**説明**
- リクエストは 1 つだけなので、答えは常に 1

### 例 3

**入力**
```
5
1000
2000
3000
4000
5000
```

**出力**
```
1 2 3 4 4
```

**説明**
- `t=1000`: `{1000}` の 1 個
- `t=2000`: `{1000, 2000}` の 2 個
- `t=3000`: `{1000, 2000, 3000}` の 3 個
- `t=4000`: `{1000, 2000, 3000, 4000}` の 4 個（`1000` はちょうど範囲内）
- `t=5000`: `{2000, 3000, 4000, 5000}` の 4 個（`1000` は範囲外）

## ヒント

- キューを使って「まだ有効なリクエスト」を管理しましょう
- 新しいリクエストが来たら、古くなったリクエストをキューから削除します
- PHP の配列を使ってキューを実装できます（`array_shift` または インデックス管理）
