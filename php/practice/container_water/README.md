# 水を貯める容器（Container With Most Water）

## 難易度
★★☆☆☆（初級〜中級）

## アルゴリズム
二つのポインタ（Two Pointers）

## 問題

高さが異なる n 本の縦線があります。
配列 `height` の `height[i]` は i 番目の線の高さを表します。

2本の線を選んで容器を作り、その容器に水を貯めます。
**貯められる水の最大量**を求めてください。

容器の水量は `min(height[i], height[j]) × (j - i)` で計算されます。

```
例: height = [1, 8, 6, 2, 5, 4, 8, 3, 7]

    |       |
    |       |   |
    | | |   |   |
    | | | | |   |
    | | | | | | |
    | | | | | | | |
  | | | | | | | | |
  0 1 2 3 4 5 6 7 8

height[1]=8 と height[8]=7 を選ぶと:
水量 = min(8, 7) × (8 - 1) = 7 × 7 = 49
```

## 入出力例

### 例1
```
入力: height = [1, 8, 6, 2, 5, 4, 8, 3, 7]
出力: 49
説明: height[1]=8 と height[8]=7 を選ぶ
```

### 例2
```
入力: height = [1, 1]
出力: 1
説明: min(1, 1) × (1 - 0) = 1
```

### 例3
```
入力: height = [4, 3, 2, 1, 4]
出力: 16
説明: height[0]=4 と height[4]=4 を選ぶ
      min(4, 4) × (4 - 0) = 16
```

### 例4
```
入力: height = [1, 2, 1]
出力: 2
説明: height[0]=1 と height[2]=1 を選ぶ
      min(1, 1) × (2 - 0) = 2
```

## 制約

- 2 <= len(height) <= 10^5
- 0 <= height[i] <= 10^4

## ヒント

### 素朴な解法（TLE）
- 全ペアを試す: O(n²)
- 制約 n ≤ 10^5 では間に合わない

### 二つのポインタ
- 左端 `left = 0`、右端 `right = n-1` から開始
- 面積を計算し、最大値を更新
- **低い方のポインタを内側に移動**
- なぜ低い方を動かす？
  - 幅は必ず狭くなる
  - 高さを上げないと面積が増える可能性がない
  - 高い方を動かしても、min は変わらないか下がるだけ
